# Streamlit 医生端网页公网部署说明

## 1. 先说结论

本项目的医生端网页是 Streamlit 应用，需要 Python 后端运行，不能用 GitHub Pages 直接部署。

推荐部署方式：

```text
GitHub 仓库 + Streamlit Community Cloud
```

部署成功后会得到一个公网链接，例如：

```text
https://你的应用名.streamlit.app
```

这个链接可以发给老师、组员或答辩现场使用。

## 2. GitHub Pages 和 Streamlit Cloud 的区别

| 平台 | 适合什么 | 是否适合本项目医生端 |
|---|---|---|
| GitHub Pages | 静态 HTML、CSS、JS 页面 | 不适合 |
| Streamlit Cloud | Python + Streamlit 数据应用 | 适合 |
| 服务器/云主机 | 长期部署、可控性高 | 适合但配置更复杂 |
| 内网穿透 | 临时演示本机网页 | 可用但不稳定 |

患者端展示页如果是纯 HTML，可以放 GitHub Pages。

医生端 `app.py` 需要运行 Python，所以建议放 Streamlit Cloud。

## 3. 当前项目已经为部署准备了什么

当前 `程序设计` 文件夹已经包含部署所需核心文件：

```text
app.py
config.py
requirements.txt
core/
data/
.streamlit/config.toml
```

其中：

| 文件/文件夹 | 作用 |
|---|---|
| `app.py` | Streamlit 网页入口 |
| `core/` | 数据处理、NNLS 分解、验证、导出等后端逻辑 |
| `config.py` | 默认路径、类别、参数配置 |
| `requirements.txt` | 云端安装依赖 |
| `data/` | 可公开部署的演示数据 |
| `.streamlit/config.toml` | Streamlit 云端运行配置 |

注意：`data/送检目录_脱敏.xlsx` 只保留 `序号 / 上理工 / 长海`，不包含姓名和住院号。

## 5. 上传到 GitHub

如果 `程序设计` 本身已经是 Git 仓库，可以在 VS Code 终端中运行：

```powershell
cd D:\生医工\机器学习\首次数据及测算\程序设计
git status
git add .
git commit -m "Prepare Streamlit public deployment"
git push
```

如果还没有绑定远程仓库，需要先添加远程地址：

```powershell
git remote add origin https://github.com/你的用户名/你的仓库名.git
git branch -M main
git push -u origin main
```

如果已经添加过 `origin`，不要重复添加。

可以用下面命令查看：

```powershell
git remote -v
```

## 6. Streamlit Cloud 部署步骤

1. 打开 Streamlit Community Cloud：

```text
https://share.streamlit.io
```

或：

```text
https://streamlit.io/cloud
```

2. 用 GitHub 登录。

3. 点击 `Create app` 或 `New app`。

4. 选择你的 GitHub 仓库。

5. 设置部署参数：

| 选项 | 应填内容 |
|---|---|
| Repository | 你的项目仓库 |
| Branch | `main` |
| Main file path | `app.py` |
| App URL | 自己起一个英文名 |

如果你的仓库不是直接以 `程序设计` 作为根目录，而是把 `程序设计` 放在仓库子文件夹里，那么 Main file path 要填：

```text
程序设计/app.py
```

但更推荐直接把 `程序设计` 文件夹里的内容作为仓库根目录。

6. 点击 Deploy。

7. 等待依赖安装完成。

部署成功后会出现公网链接。

## 7. 常见问题

### 7.1 ModuleNotFoundError

通常是依赖没写进 `requirements.txt`。

当前项目需要：

```text
streamlit
pandas
numpy
scipy
scikit-learn
plotly
openpyxl
joblib
```

这些已经写在 `requirements.txt` 中。

### 7.2 FileNotFoundError

通常是云端找不到数据文件。

请检查 GitHub 仓库里是否有 `data/` 文件夹，以及里面是否包含：

```text
data-纯品.csv
data-原始_基础展示120.csv
data-均值.csv
送检目录_脱敏.xlsx
纯品PCA分类.png
病人结石PCA投影.png
```

### 7.3 页面显示空白或部署失败

可以查看 Streamlit Cloud 的 Logs。

重点看：

- 是否安装依赖失败；
- 是否找不到文件；
- 是否中文文件名路径出错。

如果担心中文文件名导致云端异常，可以后续把 `data/` 内文件名统一改成英文。

### 7.4 能不能部署 GitHub Pages

医生端不能。

原因是 GitHub Pages 只能托管静态网页，不能运行 Python 后端。

患者端 HTML 可以部署 GitHub Pages。

## 8. 分享方式

部署成功后，把公网链接发给别人即可。

也可以把链接转成二维码：

1. 复制 Streamlit 公网链接；
2. 打开任意二维码生成器；
3. 粘贴链接；
4. 下载二维码图片；
5. 放到 PPT 或海报里。

## 10. 隐私提醒

不要把以下内容上传到公开仓库：

- 姓名；
- 住院号；
- 手机号；
- 身份证号；
- 原始病历；
- 未脱敏的送检目录；
- 任何可反查个人身份的信息。

当前部署用的 `data/送检目录_脱敏.xlsx` 已经去掉姓名和住院号。
