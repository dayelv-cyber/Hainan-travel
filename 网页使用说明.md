# 尿结石 FTIR 标准品比对与自动初筛系统说明

## 1. 这个网页是做什么的

本网页是一个尿路结石光谱分析演示系统，主要用于把已测得的结石光谱数据与纯品标准谱进行比对，输出每个样品更接近哪些成分。

系统目前定位为科研展示和教学参考，不作为临床诊断或治疗依据。页面中的概率应理解为光谱贡献比例或谱型相似贡献，不等同于严格的化学定量百分比。

当前系统主要面向 5 类成分：

| 缩写 | 中文名称 |
|---|---|
| CAOX | 草酸钙 |
| DCY | 胱氨酸 |
| URIC | 尿酸 |
| APA | 磷灰石/碳酸磷灰石 |
| DCP | 磷酸氢钙 |

## 2. 别人能不能直接看到网页

如果网页地址是 `http://localhost:8503` 或 `http://localhost:8502`，这个地址只代表“你自己电脑”。别人电脑或手机不能直接用这个地址访问。

常见访问方式有三种：

| 方式 | 能否给别人看 | 适合场景 |
|---|---|---|
| 本机 localhost | 只有自己能看 | 自己调试 UI、改代码 |
| 局域网访问 | 同一 WiFi/校园网内可能能看 | 小组内部临时演示 |
| 公网部署 | 所有人都能通过链接或二维码访问 | 答辩、展示、远程分享 |

如果要让同一 WiFi 下的同学访问，需要用你电脑的局域网 IP，例如：

```text
http://你的电脑IP:8503
```

同时要求：

- 你的电脑正在运行网页；
- 电脑不能关机或退出 Streamlit；
- 防火墙允许别人访问该端口；
- 访问者和你在同一个网络环境中。

如果要扫码给任何人访问，建议把静态患者端页面放到 GitHub Pages；而这个 Streamlit 医生端网页如果要公网访问，需要部署到 Streamlit Cloud、服务器、内网穿透或其他云平台。

## 3. 系统当前运行流程

网页默认读取 `最终文件/大创` 文件夹中的数据。也可以在左侧上传一个新的原始重复谱 CSV 临时替换默认数据。

整体流程如下：

```text
原始重复谱 CSV
  ↓
按样品号求平均谱
  ↓
与纯品标准谱统一频率轴
  ↓
NNLS 五成分分解
  ↓
计算拟合残差
  ↓
结合送检弱标签做验证
  ↓
输出主表、样品详情、PCA 图、验证结果和科研参考
```

## 4. 输入文件是什么

左侧上传区目前只需要上传一个原始重复谱 CSV。

推荐格式：

| 第一列 | 后续列 |
|---|---|
| `wave_num` 或频率/波数轴 | `S31-1`, `S31-2`, `S31-3` 等样品重复谱 |

系统会自动识别同一样品的重复谱，例如：

```text
S31-1, S31-2, S31-3, S31-4, S31-5, S31-6
```

会被平均成一个样品：

```text
S31
```

也就是说，使用者不需要手动先求平均，后端会自动处理。

## 5. 当前核心算法是什么

当前主算法是 NNLS，中文可以理解为“非负最小二乘分解”。

它的基本思想是：

```text
未知样品谱 ≈ CAOX纯品谱 × a
          + DCY纯品谱 × b
          + URIC纯品谱 × c
          + APA纯品谱 × d
          + DCP纯品谱 × e
```

其中 `a, b, c, d, e` 都不能为负数。系统会自动寻找一组最合适的非负权重，使组合后的谱图尽量接近样品谱。

输出结果可以理解为：

- 哪个纯品谱对当前样品解释最多；
- 当前样品是否可能是混合成分；
- 当前纯品库能否较好拟合这个样品。

## 6. 拟合残差是什么意思

拟合残差表示“系统用当前 5 类纯品谱去解释某个样品谱时，剩下解释不掉的差距有多大”。

简单理解：

| 残差情况 | 含义 |
|---|---|
| 拟合残差正常 | 当前纯品库对该样品谱解释较好 |
| 拟合残差偏高 | 当前纯品库解释不充分，建议复核 |

残差偏高可能来自：

- 样品确实是复杂混合成分；
- 有当前纯品库未覆盖的成分；
- 谱图噪声、基线或仪器状态影响；
- 样品制备或测量过程存在差异。

因此，残差不是“错误率”，而是一个复核提示。

## 7. 页面各部分怎么看

### 7.1 顶部指标

顶部会显示：

- 样品数：当前参与分析的有效样品数量；
- 类别数：当前纯品基底类别数量；
- 合理归属：NNLS 主成分是否落在送检详细成分中；
- 组合命中：NNLS 前二成分是否与送检详细成分有交集；
- 残差复核：拟合残差偏高、建议进一步检查的样品数。

### 7.2 本批样品结论概览

该图统计每类成分作为 NNLS 最高贡献成分的样品数。

注意：这个图是“样品个数统计”，不是整批样品的真实化学组成比例。

### 7.3 NNLS 主表

主表每一行代表一个样品。

主要列含义：

| 列名 | 含义 |
|---|---|
| 样品 | 样品编号，点击可进入该样品详情 |
| NNLS主成分 | 光谱贡献最高的成分 |
| 主成分相对贡献 | 最高成分的贡献值 |
| NNLS成分构成 | 贡献值超过阈值的主要成分组合 |
| 构成状态 | 对混合程度的文字解释 |
| 拟合残差 | 当前纯品组合对样品谱的拟合差距 |
| 残差标记 | 残差是否偏高 |
| 综合判断 | 结合 NNLS 构成和残差后的总结 |

### 7.4 验证页

验证页使用送检弱标签做参考验证。

目前使用三种口径：

| 口径 | 含义 |
|---|---|
| 严格口径 | NNLS 主成分必须等于粗标签 |
| 合理归属 | NNLS 主成分出现在详细成分中 |
| 成分组合 | NNLS 前二成分与详细成分有交集 |

混合结石中，严格口径通常偏严；合理归属和成分组合更适合解释混合样品。

### 7.5 样品详情

点击主表中的样品编号，或在样品详情页手动选择样品，可以查看：

- 该样品五成分相对贡献柱状图；
- 该样品平均谱；
- 该样品拟合残差。

柱状图颜色与 PCA 图保持一致：

| 成分 | 颜色含义 |
|---|---|
| APA | 蓝色 |
| CAOX | 橙色 |
| DCP | 绿色 |
| DCY | 红色 |
| URIC | 紫色 |

### 7.6 PCA 图

PCA 图用于展示纯品谱和病人样品谱在降维空间中的相对位置。

它主要用于可视化参考：

- 看纯品之间是否能分开；
- 看病人样品更靠近哪类纯品区域；
- 辅助理解样品可能存在的混合趋势。

PCA 图不是最终判定依据，最终主表以 NNLS 分解结果和残差质控为主。

### 7.7 科研参考

科研参考页默认不展开。点击“方法依据和参考文献”按钮后，可以查看：

- 成分解释；
- NNLS 和拟合残差的方法依据；
- 相关参考文献列表；
- 本系统如何引用这些文献作为背景说明。

## 8. 当前结果应该怎么表述

推荐表述：

```text
本系统基于纯品标准谱，采用 NNLS 方法对尿路结石样品进行五成分光谱贡献分解。
结果显示，部分样品呈现单一主成分特征，部分样品表现出明显混合谱型。
结合送检弱标签验证，系统在合理归属和成分组合口径下具有一定参考价值。
拟合残差用于提示当前纯品库对样品谱的解释充分程度，残差偏高样品建议进一步复核。
```

不推荐表述：

```text
系统可以直接诊断病人的结石成分。
```

也不推荐把概率说成：

```text
某样品含有 56.8% 草酸钙。
```

更稳妥的说法是：

```text
该样品中 CAOX 标准谱贡献最高，相对贡献为 0.5686，提示其谱型更接近草酸钙相关成分。
```

## 9. 当前局限

当前系统仍有局限：

- 样品数量有限；
- 送检标签属于弱标签，不是完整严格定量标签；
- 当前纯品库只覆盖 5 类成分；
- 概率是光谱贡献估计，不是化学百分比；
- 部分成分谱图可能存在重叠；
- 拟合残差偏高的样品需要进一步复核；
- 系统不作为临床诊断依据。

## 12. 常见问题

### Q1：为什么不能直接说是诊断结果？

因为当前数据量有限，标签也不是完整定量标签，系统输出主要是基于光谱和标准品的科研参考结果。

### Q2：为什么有些样品是混合成分明显？

因为尿路结石本身可能是混合结石，光谱上会同时体现多种成分特征。NNLS 正好适合表达这种“多个纯品谱共同贡献”的情况。

### Q3：为什么 DCY 没有样品？

这表示当前批次中没有样品以 DCY 作为最高贡献成分。但 DCY 仍保留在五类纯品库中，样品详情页也会显示其相对贡献。

### Q4：为什么需要拟合残差？

因为仅看最高贡献成分不够。拟合残差可以提示“当前 5 类纯品能不能解释这个样品谱”。如果残差偏高，即使有最高成分，也建议复核。

### Q5：如果别人要看网页怎么办？

本机调试时只能自己看。若要别人访问，需要局域网 IP、服务器部署或公网平台。手机端扫码展示建议使用 GitHub Pages 或静态网页部署。
